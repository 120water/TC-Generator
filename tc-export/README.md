# TC Export to Zephyr

HTML tool to view test cases generated by the AI and export them to CSV ready for Zephyr Scale.

---

## tc-export concept (recap)

1. **Text data only**  
   When the AI finishes creating TCs, it writes **only the values** (no repeated HTML) to a **`generated-tcs.json`** file. The HTML does not duplicate lines: there is a single container and in JS a **loop** that renders one card per TC from that JSON.

2. **Single “component” in a loop**  
   The HTML has one list block (`#tcList`). The JS iterates over the TC array and generates one card per item (same layout, no repeated lines in the HTML).

3. **Export to CSV button**  
   A button lets the user **export to .csv** the selected TCs for import into Zephyr.

4. **Selection in the UI**  
   The user chooses **which TCs** to export via the interface (checkboxes and “Select all”).

---

## Data in HTML or in a .json file?

A **separate .json file** (`generated-tcs.json`) is used:

- The AI writes only to that file (it does not modify the HTML).
- The HTML stays fixed: structure plus a single container; text comes from the JSON.
- If you open the page with **file://**, the browser cannot `fetch` the .json for security reasons. So there is a **“Load from file”** button: the user selects `generated-tcs.json` and the TCs are loaded.
- If you open the page with a **local server** (Live Server, `npx serve`, etc.) from the `tc-export` folder, the page can load `generated-tcs.json` automatically on open.

---

## Usage

1. Open `tc-export.html` in your browser.
2. **Load TCs:**
   - With a local server: `generated-tcs.json` is loaded automatically on open.
   - Without a server (double-click the HTML): use **“Load from file (generated-tcs.json)”** and select `generated-tcs.json` (where the AI wrote the TCs).
   - Alternative: **“Paste JSON (from AI)”** and paste the TC array.
3. **Add:** “Add test case” to create a TC manually.
4. **Edit:** “Update” on any TC to change name, steps, etc.
5. **Select** with the checkboxes which TCs to export (and “Select all” if desired).
6. **Export selected to CSV** downloads a CSV with the Zephyr canonical structure (§8.1).

---

## JSON format (generated-tcs.json or paste)

Array of test cases. Fields in camelCase:

```json
[
  {
    "name": "Short TC title",
    "status": "Draft",
    "precondition": "User is logged in.",
    "objective": "Verify that the button submits the form.",
    "priority": "Normal",
    "labels": "functional_test",
    "estimatedTime": "00:05",
    "automated": "No",
    "automationPossible": "Yes",
    "createdInVersion": "26.1.0.0",
    "steps": [
      { "step": "Click Submit", "testData": "", "expectedResult": "Form is submitted." },
      { "step": "Open the list", "testData": "", "expectedResult": "New record is visible." }
    ]
  }
]
```

- **labels:** one of: `functional_test`, `static_text_test`, `ui_design_test`
- **estimatedTime:** format `hh:mm`, max 00:15
- **steps:** at least one; each step: `step`, `testData` (optional), `expectedResult`

The exported CSV follows the canonical structure (Name, Status, Precondition, Objective, Priority, Labels, Estimated Time, Automated, Automation Possible, Created in Version, Step, Test Data, Expected Result), one row per step, UTF-8 with BOM.
